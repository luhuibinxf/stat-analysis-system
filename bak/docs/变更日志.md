## 1. 数据库连接加密方案

### 1.1 加密算法
- **算法**: AES-128-CBC
- **密钥长度**: 128位 (16字节)
- **初始向量**: 128位 (16字节)
- **填充方式**: PKCS7
- **输出编码**: Base64

### 1.2 支持的密钥列表

| 序号 | 密钥 (十六进制) | 说明 |
|------|---------------|------|
| 1 | `12-34-56-78-90-AB-CD-EF-12-34-56-78-90-AB-CD-EF` | 默认密钥 |
| 2 | `00-01-02-03-04-05-06-07-08-09-0A-0B-0C-0D-0E-0F` | 顺序密钥 |
| 3 | `FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF` | 全1密钥 |
| 4 | `41-42-43-44-45-46-47-48-49-4A-4B-4C-4D-4E-4F-50` | ASCII顺序 |
| 5 | `01-12-23-34-45-56-67-78-89-9A-AB-BC-CD-DE-EF-F0` | 递增密钥 |
| 6 | `11-22-33-44-55-66-77-88-99-AA-BB-CC-DD-EE-FF-00` | 重复密钥 |

### 1.3 配置文件处理流程

```
启动程序
    │
    ├─ 检查 config.dat 是否存在
    │       │
    │       ├─ 存在 → 读取内容
    │       │       │
    │       │       ├─ 尝试用6种密钥解密
    │       │       │       │
    │       │       │       ├─ 解密成功 → 使用连接字符串
    │       │       │       │
    │       │       │       └─ 解密失败 → 检查是否为明文
    │       │       │                   │
    │       │       │                   ├─ 是明文 → 直接使用
    │       │       │                   │
    │       │       │                   └─ 不是明文 → 弹出配置窗口
    │       │       │
    │       │       └─ 用户输入后 → AES加密保存
    │       │
    │       └─ 不存在 → 弹出配置窗口
    │                   │
    │                   └─ 用户输入后 → AES加密保存
```

## 6. 代码规范

### 6.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `ConnectionStrings` |
| 方法名 | PascalCase | `GetConnectionString` |
| 变量名 | camelCase | `connectionString` |
| 常量名 | UPPER_CASE | `DEFAULT_PORT` |
| 文件命名 | PascalCase | `DailyAnalysisService.cs` |

### 6.2 错误处理规范

1. 所有数据库操作必须使用 try-catch 包裹
2. 捕获异常时记录日志 (`LogHelper.LogError`)
3. 数据库连接失败时提供友好的错误提示
4. 配置文件读取失败时使用默认值或弹出配置窗口

### 6.3 日志规范

| 日志级别 | 使用场景 | 示例 |
|----------|---------|------|
| INFO | 正常业务流程 | "HTTP服务器启动成功" |
| ERROR | 异常情况 | "数据库连接失败" |
| DEBUG | 调试信息 | "配置文件路径: xxx" |
