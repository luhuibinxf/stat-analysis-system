## 1. 项目概述

### 1.1 项目目标
开发一个基于C# Windows Forms的统计分析系统，提供医院检查数据的统计分析功能。

### 1.2 核心功能
- 每日检查分析
- 科室统计
- 医生工作量统计
- 检查类型统计
- 阴阳性率统计

### 1.3 技术栈
| 分类 | 技术 | 版本 |
|------|------|------|
| 语言 | C# | .NET 8.0 |
| 框架 | Windows Forms | - |
| 数据库 | SQL Server | 2019+ |
| HTTP服务器 | HttpListener | 内置 |
| 前端 | HTML/CSS/JavaScript | - |

## 3. 数据库连接加密方案

### 3.1 加密算法

| 参数 | 值 |
|------|-----|
| 算法 | AES-128-CBC |
| 密钥长度 | 128位 (16字节) |
| 初始向量(IV) | 128位 (16字节) |
| 填充方式 | PKCS7 |
| 输出编码 | Base64 |

### 3.2 密钥配置

**默认密钥**:
```csharp
byte[] _encryptionKey = { 
    0x12, 0x34, 0x56, 0x78, 0x90, 0xAB, 0xCD, 0xEF, 
    0x12, 0x34, 0x56, 0x78, 0x90, 0xAB, 0xCD, 0xEF 
};
```

**支持的6种解密密钥**:
| 序号 | 密钥 (十六进制) |
|------|---------------|
| 1 | `12-34-56-78-90-AB-CD-EF-12-34-56-78-90-AB-CD-EF` |
| 2 | `00-01-02-03-04-05-06-07-08-09-0A-0B-0C-0D-0E-0F` |
| 3 | `FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF-FF` |
| 4 | `41-42-43-44-45-46-47-48-49-4A-4B-4C-4D-4E-4F-50` |
| 5 | `01-12-23-34-45-56-67-78-89-9A-AB-BC-CD-DE-EF-F0` |
| 6 | `11-22-33-44-55-66-77-88-99-AA-BB-CC-DD-EE-FF-00` |

### 3.3 配置文件处理流程

```
启动程序
    │
    ├─ 查找 config.dat 文件
    │       │
    │       ├─ 文件存在 → 读取内容
    │       │       │
    │       │       ├─ 尝试用6种密钥解密
    │       │       │       │
    │       │       │       ├─ 解密成功 → 使用连接字符串
    │       │       │       │
    │       │       │       └─ 解密失败 → 检查是否为明文
    │       │       │                   │
    │       │       │                   ├─ 是明文 → 直接使用
    │       │       │                   │
    │       │       │                   └─ 不是明文 → 弹出配置窗口
    │       │       │
    │       │       └─ 用户输入后 → AES加密保存
    │       │
    │       └─ 文件不存在 → 弹出配置窗口
    │                   │
    │                   └─ 用户输入后 → AES加密保存
```

### 3.4 加密/解密示例

**加密过程**:
```csharp
string connectionString = "Server=localhost;Database=TestDB;User ID=sa;Password=123456;";
string encrypted = AesEncrypt(connectionString, key, key);
// 输出: Base64编码的加密字符串
```

**解密过程**:
```csharp
string encrypted = "kkzI4Gy8No7vpAZuMSLF4rfumYQ03P4PVD1O2DTNq2eqLHxmYb3uXVRWRC1d9NHnFbIUgN4RFN3Qr3FliE17T0u4oO1cwudk3U2Y0Wuxj/0=";
string decrypted = AesDecrypt(encrypted, key, key);
// 输出: 原始连接字符串
```

## 6. 问题处理办法

### 6.1 常见问题及解决办法

| 问题 | 现象 | 原因 | 解决办法 |
|------|------|------|---------|
| 端口被占用 | 启动失败，提示端口被占用 | HTTP.sys服务占用端口 | 使用管理员权限启动，或修改默认端口 |
| 数据库连接失败 | 登录时提示连接失败 | 配置文件无效或数据库未启动 | 删除config.dat，重新配置 |
| JSON解析失败 | 接口返回错误 | Content-Type未设置 | 确保设置application/json头 |
| 配置文件解密失败 | 日志显示解密失败 | 密钥不匹配 | 删除config.dat，重新配置 |
| 医院名称不显示 | 登录界面显示XX医院 | 数据库查询失败 | 检查数据库连接和查询语句 |

### 6.2 问题排查流程

```
问题发生
    │
    ├─ 查看日志 (server.log)
    │       │
    │       ├─ 端口占用 → 检查端口占用情况
    │       │       │
    │       │       └─ netstat -ano | findstr :9094
    │       │
    │       ├─ 数据库连接失败 → 检查配置文件
    │       │       │
    │       │       └─ 删除config.dat重新配置
    │       │
    │       └─ 解密失败 → 删除config.dat重新配置
    │
    └─ 检查服务状态
            │
            └─ Get-Process | Where-Object { $_.ProcessName -like '*DbProcedureCaller*' }
```

### 6.3 常用命令

| 命令 | 用途 | 示例 |
|------|------|------|
| 查看端口占用 | 检查端口是否被占用 | `netstat -ano | findstr :9094` |
| 查看服务进程 | 检查服务是否运行 | `Get-Process | Where-Object { $_.ProcessName -like '*DbProcedureCaller*' }` |
| 停止服务 | 强制停止服务进程 | `Stop-Process -Id <PID> -Force` |
| 查看日志 | 查看最新日志 | `Get-Content server.log | Select-Object -Last 20` |
| 删除配置 | 删除加密配置文件 | `Remove-Item config.dat -Force` |

| server_config.dat | d:\AI\tran\server_config.dat | 服务器配置（端口等） | 明文 |
| server.log | 程序运行目录 | 运行日志 | 文本 |

### 8.2 config.dat 说明

**用途**: 存储加密后的数据库连接字符串

**格式**: AES-128-CBC加密后Base64编码

**查找顺序**:
1. 程序运行目录
2. `d:\AI\tran\config.dat`
3. 项目根目录（相对路径）

**修改方法**:
1. 删除 `config.dat` 文件
2. 重新启动程序
3. 在弹出的配置窗口中输入数据库连接信息
4. 点击"测试连接"验证
5. 点击"确定"保存（自动加密）

### 8.3 server_config.dat 说明

**用途**: 存储服务器配置参数

**格式**: 明文键值对

**内容示例**:
```
Port=9094
```

### 8.4 server.log 说明

**用途**: 记录程序运行日志

**格式**: 文本文件，每行一条日志

**日志级别**:
- `[INFO]`: 正常业务流程
- `[ERROR]`: 异常情况
