# WiNEX 统计验证 Agent 使用指南

## Agent 信息

- **名称**：winex-stat-validator
- **版本**：v1.0.0
- **创建日期**：2026-04-23
- **功能**：WiNEX检查统计验证

---

## 快速开始

### 1. 启动 Agent

在 Trae IDE 中切换到此 Agent：
```
切换到 winex-stat-validator Agent
```

### 2. 验证数据库连接

首次使用时请先验证数据库连接：
```powershell
SQLCMD.EXE -S localhost -d WiNEX_PACS -Q "SELECT DB_NAME()" -W
```

## 使用示例

### 放射科工作量统计

#### 请求：统计4月1日的放射科工作量
```
帮我统计4月1日的放射科工作量
```

#### Agent 执行：
1. 连接数据库
2. 执行总数据量查询
3. 执行各检查类别查询
4. 汇总各组数据
5. 验证数据准确性
6. 生成报告

#### 输出：
```
=== WiNEX 统计验证报告 ===

统计日期：2026-04-01
统计类型：放射科（RIS）

【总数据】
总检查数：539条

【各组数据】
普放组：186条（DR:180 + 钼靶:3 + 造影:3）
CT组：270条（CT平扫:229 + CT增强:41）
MRI组：83条（MRI平扫:83）

【验证结果】
✓ 普放组 + CT组 + MRI组 = 539条
✓ 验证通过

【各医生工作量】
1. 张银川：45条
2. 周亮：38条
3. ...
```

### 内镜科工作量统计

#### 请求：验证4月2日的内镜科医生工作量
```
验证4月2日内镜科医生工作量
```

#### Agent 执行：
1. 连接数据库
2. 执行胃镜数据查询
3. 执行肠镜数据查询
4. 验证：胃镜 = 无痛胃镜 + 普通胃镜
5. 验证：肠镜 = 无痛肠镜 + 普通肠镜
6. 生成报告

#### 输出：
```
=== WiNEX 内镜科统计报告 ===

统计日期：2026-04-02
统计类型：内镜科（EIS）

【总数据】
总检查数：120条

【胃镜统计】
胃镜总数：80条
- 无痛胃镜：60条
- 普通胃镜：20条
✓ 胃镜 = 无痛胃镜 + 普通胃镜（60+20=80）

【肠镜统计】
肠镜总数：40条
- 无痛肠镜：30条
- 普通肠镜：10条
✓ 肠镜 = 无痛肠镜 + 普通肠镜（30+10=40）

【验证结果】
✓ 所有验证通过
```

### 数据差异分析

#### 请求：检查4月3日的数据差异
```
检查4月3日的数据差异
```

#### Agent 执行：
1. 连接数据库
2. 查询统计数据
3. 执行验证公式
4. 发现差异时输出分析

#### 输出（发现差异时）：
```
=== 数据差异分析 ===

日期：2026-04-03
组别：普放组

【差异详情】
预期：186条
实际：180条
差异：6条

【可能原因】
部分检查类别未被识别

【建议】
检查EXAM_CATEGORY_NAME字段，补充遗漏的检查类型
```

## 支持的统计类型

### 放射科（RIS）
| 组别 | 包含检查类别 |
|------|-------------|
| 普放组 | 普放、普放(新)、钼靶、消化道造影 |
| CT组 | CT、CT(新)、CT增强、CTA等 |
| MRI组 | 核磁共振、MRI、MRA、MRV等 |

### 内镜科（EIS）
| 组别 | 包含检查项目 |
|------|-------------|
| 胃镜 | 胃镜、胃+肠、普通胃镜、无痛胃镜 |
| 肠镜 | 肠镜、普通肠镜、无痛肠镜 |
| 其他 | 超声胃镜、超声肠镜、支气管镜、ERCP |

---

## 验证公式

### 放射科
- 普放组 = DR + 钼靶 + 造影 + 下肢及脊柱全长拍片
- CT组 = CT平扫 + CT增强
- MRI组 = MRI平扫 + MRI增强 + MRA + MRV + MRU + MRCP + MRS
- **总数据 = 普放组 + CT组 + MRI组**

### 内镜科
- 胃镜 = 无痛胃镜 + 普通胃镜
- 肠镜 = 无痛肠镜 + 普通肠镜

## 配置文件

Agent 配置文件位于：
```
.trae/agents/winex-stat-validator/
├── AGENT.md      # Agent 主配置文件
├── config.json   # 数据库连接配置
└── rules.json    # 统计规则配置
```

### config.json 配置项
```json
{
  "database": {
    "server": "localhost",      // SQL Server 地址
    "database": "WiNEX_PACS",   // 数据库名称
    "authentication": "windows"  // 认证方式
  }
}
```

### rules.json 配置项
定义各检查类型的分类规则和验证公式。
