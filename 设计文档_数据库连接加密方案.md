# 数据库连接配置加密方案设计文档

## 1. 概述

本系统采用 AES-128-CBC 加密算法对数据库连接字符串进行加密存储，确保敏感配置信息的安全性。

## 2. 加密方案

### 2.1 加密算法

| 参数 | 值 |
|------|-----|
| 算法 | AES-128-CBC |
| 密钥长度 | 128位 (16字节) |
| 初始向量(IV) | 128位 (16字节) |
| 填充方式 | PKCS7 |
| 输出编码 | Base64 |

### 2.2 密钥配置

```csharp
// 加密密钥 (16字节)
private static readonly byte[] _encryptionKey = new byte[] { 
    0x12, 0x34, 0x56, 0x78, 0x90, 0xAB, 0xCD, 0xEF, 
    0x12, 0x34, 0x56, 0x78, 0x90, 0xAB, 0xCD, 0xEF 
};

// 初始向量 (16字节)
private static readonly byte[] _iv = new byte[] { 
    0x12, 0x34, 0x56, 0x78, 0x90, 0xAB, 0xCD, 0xEF, 
    0x12, 0x34, 0x56, 0x78, 0x90, 0xAB, 0xCD, 0xEF 
};
```

### 2.3 加密流程

```
明文连接字符串 → AES-128-CBC加密 → Base64编码 → 写入 config.dat
```

### 2.4 解密流程

```
读取 config.dat → Base64解码 → AES-128-CBC解密 → 明文连接字符串
```

## 3. 配置文件格式

### 3.1 文件路径

配置文件 `config.dat` 存放位置（按优先级）：
1. 程序运行目录
2. `d:\AI\tran\config.dat`
3. 项目根目录

### 3.2 文件格式

配置文件为单行文本文件，内容为经过 **AES-128-CBC + Base64** 加密后的数据库连接字符串。

**示例：**
```
kkzI4Gy8No7vpAZuMSLF4rfumYQ03P4PVD1O2DTNq2eqLHxmYb3uXVRWRC1d9NHnFbIUgN4RFN3Qr3FliE17T0u4oO1cwudk3U2Y0Wuxj/0=
```

## 4. 当前问题分析

### 4.1 问题描述

现有配置文件中的加密数据无法被正确解密，原因是：
- 当前代码使用的密钥与原始加密时使用的密钥不一致
- 配置文件中的数据是使用**不同密钥**加密的

### 4.2 解决方案

**方案一：找回原始密钥**
- 查找历史代码或文档中记录的原始加密密钥
- 使用原始密钥进行解密

**方案二：重新加密**
- 删除旧的配置文件
- 用户重新输入数据库连接信息
- 使用当前密钥重新加密并保存

**方案三：支持多种密钥**
- 尝试多种可能的密钥进行解密
- 优先使用当前密钥，失败时提示用户配置

## 5. API接口文档

### 5.1 医院信息接口

**接口地址：** `GET /get-hospital-info`

**功能描述：** 从数据库获取医院名称

**请求参数：** 无

**响应格式：**
```json
{
    "success": true,
    "hospitalName": "XX医院"
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 请求是否成功 |
| hospitalName | string | 医院名称（数据库获取或默认值） |

### 5.2 每日分析接口

**接口地址：** `POST /daily-analysis`

**功能描述：** 获取每日检查统计分析数据

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| startDate | string | 否 | 开始日期（支持today/yesterday/thisweek等） |
| endDate | string | 否 | 结束日期 |
| system | string | 否 | 系统类型（RIS/UIS/EIS） |
| reporter | string | 否 | 报告医生（支持多选，逗号分隔） |
| reviewer | string | 否 | 审核医生（支持多选，逗号分隔） |
| technician | string | 否 | 技师（支持多选，逗号分隔） |
| department | string | 否 | 执行科室（支持多选，逗号分隔） |
| category | string | 否 | 检查类型（支持多选，逗号分隔） |
| patientType | string | 否 | 病人类型（支持多选，逗号分隔） |
| resultStatus | string | 否 | 结果状态（阳性/阴性/未知，支持多选） |
| sortBy | string | 否 | 排序字段 |
| sortOrder | string | 否 | 排序方向（ASC/DESC） |
| pageSize | int | 否 | 每页数量（0表示不分页） |
| pageIndex | int | 否 | 页码 |

**响应格式：**

{
    "success": true,

## 7. 安全建议

### 7.1 密钥管理
- 密钥应存储在安全的位置，不应硬编码在代码中
- 考虑使用密钥管理服务或环境变量存储密钥

### 7.2 配置文件安全
- 配置文件应设置适当的文件权限，禁止未授权访问
- 传输过程中应使用 HTTPS 协议

### 7.3 连接池管理
- 使用连接池复用数据库连接
- 设置合理的连接超时时间
- 定期清理无效连接
